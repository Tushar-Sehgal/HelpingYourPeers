Interface Injection

Interface Injection involves providing dependencies through an interface that the class implements. This method is less common and often considered less flexible.

Advantages

  • Provides a standard method for injecting dependencies.
  • Ensures that dependencies are provided through a well-defined interface.

Disadvantages

  • Less flexible than other methods of dependency injection.
  • Can lead to tight coupling between the service and the dependency.

Example in Java

public interface IService {
    void setRepository(IRepository repository);
}

public class Service implements IService {
    private IRepository repository;

    @Override
    public void setRepository(IRepository repository) {
        this.repository = repository;
    }

    public void execute() {
        repository.save();
    }
}
